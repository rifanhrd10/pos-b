### Database Schema (New Models)

```prisma
model Category {
  id         String    @id @default(cuid())
  businessId String
  name       String
  description String?
  icon       String?
  sortOrder  Int       @default(0)

  @@unique([businessId, name])

model Product {
  id          String   @id @default(cuid())
  businessId  String
  categoryId  String?
  sku         String?
  barcode     String?
  name        String
  description String?
  image       String?
  basePrice   Float    @default(0)
  costPrice   Float?
  taxRate     Float    @default(0)
  isActive    Boolean  @default(true)

  sortOrder   Int      @default(0)
  createdAt   DateTime @default(now())

  business   Business          @relation(fields: [businessId], references: [id], onDelete: Cascade)
  category   Category?         @relation(fields: [categoryId], references: [id])
  variants   ProductVariant[]
  toppings   ProductTopping[]
}

model ProductVariant {
  id              String  @id @default(cuid())
  productId       String
  name            String  // "Size S", "Size M", "Reguler", "Large"
  priceAdjustment Float   @default(0)
  stock           Int     @default(0)
  isActive        Boolean @default(true)
  sortOrder       Int     @default(0)

  product Product @relation(fields: [productId], references: [id], onDelete: Cascade)
}

model ProductTopping {
  id        String  @id @default(cuid())
  productId String
  name      String  // "Extra Shot", "Boba", "Cheese"
  price     Float   @default(0)
  isActive  Boolean @default(true)

  product Product @relation(fields: [productId], references: [id], onDelete: Cascade)
}
```

**Flow rekomendasi:**

```
1. User buka /products/new
2. Baca Business.type dari session
3. Fetch: templates.ts (built-in) + Gemini recommendation (async, optional)
4. Tampilkan panel rekomendasi:
   - "Rekomendasi untuk Coffee Shop Anda"
   - Grid item dengan centang checkbox
   - Tiap item: [x] nama - Rp harga → kategori
5. User centang yang diinginkan + klik "Tambahkan"
6. Bulk create products
7. Jika Gemini gagal: fallback ke template built-in tanpa error
```

### OCR — Photo Menu to Products

#### Flow

```
1. User klik "Foto Menu" di halaman products/new
2. Camera upload / file picker (gambar)
3. File diupload ke server route handler
4. Route panggil Gemini Vision dengan prompt:
   "Extract menu items from this image. Return JSON array..."
5. Gemini return JSON draft products
6. Tampilkan hasil sebagai "Hasil scan menu" — card list yang bisa diedit
7. User hapus item yang tidak sesuai, edit nama/harga
8. Klik "Simpan" → create products ke DB
```

#### Prompt Design

```text
You are a menu parser. Analyze this restaurant/food business menu image.
Extract ALL items and return JSON:
[
  {
    "name": "Nama Produk",
    "basePrice": 35000,
    "description": "Short description if visible",
    "categoryName": "Category inferred"
  }
]
Only return the JSON array. No explanation. No markdown wrapping.
If you cannot read the image clearly, return {"error": "Gambar tidak jelas, silakan upload ulang"}
```

#### Safety

- User **wajib review** sebelum save — tidak auto-commit
- Ada loading state + "Retry dengan foto lebih jelas" button
- Jika Gemini error/rate limit → fallback ke template recommendation biasa
- Ukuran file max: 5MB, format: jpg/png/webp

## Error Handling Strategy

| Component | Error | Handling |
|-----------|-------|----------|
| Gemini API | Rate limit (429) | Fallback ke templates.ts, toast "Rekomendasi AI sedang sibuk, pakai template default" |
| Gemini API | Invalid response | Retry 1x, fallback templates.ts |
| Image upload | File terlalu besar | Reject + toast "Max 5MB" |
| Image upload | Format tidak support | Reject + validasi client side |
| Product create | Zod validation error | Return error per field |
| Permission | No access | Redirect atau show "Tidak punya akses" |
| Outlet context | Invalid outletId | Fallback ke null (All Outlets) |
